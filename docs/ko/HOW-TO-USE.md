## 새 문서 작성하기

자연어로 지시하면 됩니다. 에이전트는 **한국어와 영어 모두** 지원합니다.

### 간단한 초안 (전면 지원)

이러한 문서 유형은 지시사항만으로 작성 가능합니다:

> "이사회 결의서 작성해줘. 신규 대표이사 선임 건."

> "컴플라이언스 가이드라인 초안 써줘. 내부자 거래 관련."

> "Draft board resolutions for the appointment of a new CEO."

### 조건부 초안 (근거 자료 필요)

의견서, 소송문서, 규제문서의 경우 **근거 자료(authority packet)** — 법령, 판례, 사실관계 — 가 필요합니다:

> "법률의견서 작성해줘. 아래 법령과 판례를 근거로." (이후 소스 제공)

> "준비서면 써줘. 관련 판례는 다음과 같아: ..."

근거 자료 없이 요청하면 **스켈레톤 초안** — 전체 구조와 상투적 표현은 있지만, 실질적 내용은 `[Authority needed: ...]`, `[Argument: ...]` 등의 자리 표시자로 채워진 문서 — 을 생성합니다.

### 내부 동작 과정

에이전트가 6단계 파이프라인(D1~D6)을 실행합니다:

1. **요청 해석** — 문서 유형, 언어, 관할권, 규범 판별
2. **구조 기획** — 규범 선택, 개요 생성
3. **섹션별 작성** — 문서 전체에 걸쳐 용어 일관성 유지
4. **자체 검토** — 8개 항목 일관성 체크리스트 (용어, 번호 매기기, 상호참조, 어체, 규범)
5. **서식 적용** — 최종 문서 서식 지정
6. **자동 저장** — `output/documents/`에 버전 관리(`_v1`, `_v2`, ...)와 함께 저장

## 기존 문서 수정하기

문서를 `input/` 폴더에 넣고, 수정 사항을 알려주세요:

> "input 폴더에 넣은 의견서 수정해줘. 3번 쟁점 부분 논거를 보강하고 결론 다시 써줘."

> "Revise the brief in the input folder. Strengthen the argument in Section III and rewrite the conclusion."

에이전트가 7단계 수정 파이프라인(R1~R7)을 실행합니다:

- 문서 구조 읽기 및 분석
- 수정 지시를 특정 섹션에 매핑
- **변경 추적 결과물**과 함께 수정 실행:
  - 가능하면 `.docx` 기본 변경 추적
  - 그렇지 않으면 redline diff, clean copy, change map 형태의 Level B 결과물
  - `.md` 출력 시 인라인 diff
- 10개 항목 일관성 검사
- `_revised_` 접미사와 자동 버전 관리로 저장

### 지원 파일 형식

| 형식 | 읽기 | 쓰기 |
|------|:---:|:---:|
| `.docx` | 가능 | 가능 |
| `.pdf` | 가능 | 가능 |
| `.md` | 가능 | 가능 |
| `.txt` | 가능 | 가능 |

## Library로 더 좋은 결과 얻기

에이전트는 `/library/`에 조직의 실제 자료를 채워둘수록 더 좋은 결과를 냅니다. `/library/`의 모든 자료는 로컬에 저장되며, ingest 소스 레지스트리도 기본적으로 로컬에만 유지됩니다.

### 무엇을 넣을 수 있나

| Library 폴더 | 넣을 자료 | 효과 |
|-------------|---------|------|
| `library/precedents/` | 잘 작성된 샘플 문서 | 문체, 구조, 표현이 팀 스타일에 맞춰짐 |
| `library/templates/` | 문서 골격 | 섹션 흐름이 안정적으로 유지됨 |
| `library/house-styles/` | 내부 스타일 규칙 | 조직 고유의 서식 적용 |
| `library/grade-a/` | 법령, 공식 가이드라인 | 조건부 문서의 근거 자료로 활용 가능 |
| `library/grade-b/` | 판례, 로펌 해설 | 2차 참고자료로 활용 |
| `library/grade-c/` | 학술 논문, 참고자료 | 배경 참고용 |

### Ingest로 소스 추가하기

법령, 가이드라인, 판례, 학술 논문을 에이전트의 참조 기반에 추가하고 싶다면 **ingest** 시스템을 사용하세요.

#### Step 1: 파일 넣기

아무 파일(PDF, DOCX, PPTX, XLSX, HTML, MD, TXT)을 `library/inbox/`에 넣습니다.

> **참고:** `.hwp` 파일은 직접 지원되지 않습니다. PDF 또는 DOCX로 변환 후 넣어주세요.

#### Step 2: 에이전트에 알리기

> "inbox에 파일 넣었어, ingest 해줘"

또는:

> "/ingest"

#### Step 3: 완료

에이전트가 자동으로:

1. **변환** — 파일을 Markdown으로 변환 (MarkItDown 사용)
2. **자동 분류** — 신뢰도 등급 판별:
   - **Grade A** — 공식 1차 소스: 법령(법률 제XXXXX호), 정부 가이드라인, 공식 기관 발간물
   - **Grade B** — 검증된 2차 소스: 판례, 로펌 뉴스레터, 변호사회 자료
   - **Grade C** — 학술/참고: 학술지 논문, 학위논문
   - **Grade D** — 거부: 뉴스 기사, AI 요약, 위키 콘텐츠 (경고 메시지 표시)
3. **메타데이터 생성** — 제목, 문서 유형, 발행처, 날짜, 키워드, 관련 법조문
4. **배치** — 적절한 `library/grade-x/` 하위 폴더에 파일 저장
5. **인덱스 업데이트** — 소스 레지스트리 갱신

새 소스가 바로 향후 초안의 참고자료로 활용될 수 있습니다.

#### 분류 예시

| 추가한 자료 | 처리 결과 |
|-----------|---------|
| 개인정보보호법 전문 (PDF) | Grade A, `grade-a/statutes/`에 배치 |
| 대법원 판결문 | Grade B, `grade-b/court-precedents/`에 배치 |
| 김앤장 뉴스레터 | Grade B, `grade-b/law-firm/`에 배치 |
| 학술 논문 (법학 저널) | Grade C, `grade-c/academic/`에 배치 |
| 뉴스 기사 | Grade D — 거부 (경고 메시지 표시) |

## 검토 강도

에이전트의 자체 검토 수준을 조절할 수 있습니다:

| 수준 | 동작 | 사용 시점 |
|------|------|---------|
| **Light** (가볍게) | 1회 검토, 치명적 문제만 | 빠른 초안, 간단한 문서 |
| **Standard** (표준) | 2회 검토, 치명적 + 주요 문제 | 기본값 — 대부분의 업무 |
| **Thorough** (꼼꼼하게) | 3회 검토, 모든 문제 | 중요 문서, 최종 제출본 |

에이전트가 문맥에서 추론합니다:

- "빨리 초안만 줘" → Light
- "최종본이야, 꼼꼼하게 봐줘" → Thorough
- 별도 언급 없음 → Standard

## 이중 규범 적용

에이전트는 **문서 언어에 따라 다른 규범을 적용**합니다:

### 한국어 문서

- 어체: 문어체 (~한다, ~하여야 한다)
- 번호: 조·항·호·목 (제1조 → ① → 1. → 가.)
- 인용: 「법률명」 제N조, 대법원 YYYY. MM. DD. 선고 판결
- 지면: A4, 바탕체 12pt (본문), 맑은 고딕 (제목)
- 예외: 한국어 법률의견서 / 클라이언트 메모는 `docs/ko-legal-opinion-style-guide.md`의 전용 프로필(여백 2.54cm, 본문 11pt, 줄간격 1.15)을 따를 수 있습니다

### 영어 문서

- 어체: 공식 법률 문체
- 번호: Article/Section/(a)/(i) (US), Clause/1.1/(a) (UK)
- 인용: Bluebook (US), OSCOLA (UK), Neutral (International)
- 지면: US Letter (US) 또는 A4 (UK/International)

에이전트가 문서 언어와 관할권에 따라 규범 세트를 자동 선택합니다. **규범 혼용은 품질 결함으로 처리됩니다.**

## 좋은 결과를 위한 팁

### 구체적으로 지시하기

| 이렇게 말고 | 이렇게 |
|-----------|--------|
| "의견서 써줘" | "임대차 분쟁에 대한 법률의견서 써줘. 근거 법령은 아래와 같아: ..." |
| "Draft a memo" | "Draft an internal memo on the regulatory implications of the new AI Act for our compliance team" |

### 좋은 선례(Precedent) 제공하기

더 좋은 결과를 얻는 가장 효과적인 방법은 `library/precedents/`에 고품질 샘플 문서를 넣는 것입니다. 목표 문서 유형과 가까운 샘플일수록 결과가 좋습니다.

### 반복 업무에 Library 활용하기

팀이 같은 유형의 문서를 자주 작성한다면:
1. `library/templates/`에 템플릿을 넣어 일관된 구조 확보
2. `library/house-styles/`에 스타일 규칙을 넣어 서식 통일
3. `library/precedents/`에 선례 문서를 넣어 문체와 표현 통일

### 범위 이해하기

이 에이전트는 **비계약** 법률문서를 작성·수정합니다. 다음은 거절됩니다:
- 계약서 작성 또는 검토 ("NDA 만들어줘" → 범위 밖)
- 법률 자문 또는 리스크 평가
- 법률 조사 (법령, 판례 검색)
- 문서 검토 또는 정확성 감사

## 이 도구가 하지 않는 것

- **법률 자문을 제공하지 않습니다.** 법률문서를 더 빠르게 작성하도록 도와주는 작성 보조 도구입니다. 결과물은 반드시 법률 전문가가 검토해야 합니다.
- **법률 조사를 하지 않습니다.** 사용자가 제공한 참조자료를 반영하지만, 법령이나 판례를 직접 검색하지 않습니다.
- **인용을 검증하지 않습니다.** 사용자가 제공한 인용은 그대로 포함되고, 빠진 것은 자리 표시자가 됩니다.
- **계약서를 작성하지 않습니다.** NDA, 라이선스 계약 등 계약서는 범위 밖입니다.
- **조직 내부 정책을 알지 못합니다** — Library를 통해 추가하지 않는 한.

---

> **기억하세요:** 이것은 자동 조종이 아니라 파워 도구입니다. 법률문서 작성을 극적으로 빠르게 만들어주지만, 최종 판단은 항상 자격을 갖춘 사람의 몫입니다.
