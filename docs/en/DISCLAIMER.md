## AI Limitations — Hallucinations and Errors

This project is powered by a large language model (Claude). LLMs can and do produce **hallucinations, factual errors, and incorrect legal writing**. Outputs may:

- Apply incorrect legal writing conventions for the target jurisdiction or document type
- Produce inconsistent terminology, numbering, or cross-references despite built-in checks
- Generate structurally plausible but substantively flawed legal prose
- Fail to follow user instructions precisely, especially for complex or multi-part revision requests
- Insert content that does not reflect the user's intent or the applicable legal framework

While this agent includes built-in **self-review pipelines** (consistency checkers, convention validators, term registries), these safeguards reduce — not eliminate — the risk of error. **A qualified human must always review and verify every output** before any reliance or action.

## No Citation Fabrication — But Placeholders Require Verification

This agent is designed to **never fabricate legal citations**. When a citation is needed but not provided by the user, the agent inserts placeholders such as `[Citation needed: description]` or `[Authority needed: description]`. However:

- User-provided citations are included verbatim — the agent does not verify their accuracy
- Placeholders must be filled and verified by a qualified professional before the document is used
- The agent's convention-based formatting of citations (Bluebook, OSCOLA, Korean conventions) does not guarantee the cited authority exists or is current

## Source Library and Ingest System

The `/library/` system and `/ingest` command allow users to add external source materials (statutes, guidelines, case law, academic papers) to the agent's reference base. Important caveats:

- **Automated classification**: The ingest system auto-classifies sources into Grades A–D based on content analysis. This classification is heuristic and may be incorrect — a secondary source could be misgraded as primary, or vice versa
- **Format conversion**: Files are converted to Markdown via automated pipelines. Conversion may introduce formatting errors, lose table structures, or misparse complex layouts
- **No independent verification**: The agent treats ingested sources as provided. It does not verify the authenticity, accuracy, or currency of user-supplied materials
- **Grade D rejection**: News articles, AI-generated summaries, and wiki content are automatically rejected. However, the classifier may occasionally accept low-quality sources or reject legitimate ones

## Data Security and Privacy

All processing runs **locally on your filesystem** — no external servers or databases are involved. However, the agent makes LLM API calls, which means **document text and your instructions are sent to the model provider** (Anthropic).

If you intend to use this for real legal work with confidential documents:

- **At minimum**, use a **commercial API key** (not a free-tier or consumer plan) so that standard commercial data handling terms apply.
- **Ideally**, use an **Enterprise plan with a Zero Data Retention (ZDR) agreement**. Note that ZDR is **not a complete guarantee** — your data still passes through the provider's infrastructure. Review Anthropic's official documentation on [ZDR and data handling](https://code.claude.com/docs/en/zero-data-retention) to determine whether it meets your organization's requirements.
- **Always** check with your organization's **security, compliance, and IT departments** before processing any confidential, privileged, or personally identifiable information through any AI tool.
