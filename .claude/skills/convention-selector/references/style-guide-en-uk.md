# UK English Legal Writing Style Guide (style-guide-en-uk)

## Scope
Use this guide for England-and-Wales style legal writing, UK-facing corporate documents, court submissions, advice memoranda, and documents that should follow UK English spelling and OSCOLA citation practice.

## Core Positioning
- Prefer plain, modern legal English over archaic drafting
- Use UK terminology consistently: `clause`, `schedule`, `the court`, `claimant`, `defendant`, `solicitor`, `counsel`
- Use UK spelling consistently: `organisation`, `authorisation`, `licence` (noun), `license` (verb), `defence`, `judgement` only where jurisdiction-specific usage requires it
- Keep tone formal, but avoid unnecessary flourish or verbosity

## Key Differences from US

| Feature | UK Convention |
|---|---|
| Terminology | "Clause" not "Section"; "Schedule" not "Exhibit" |
| Page size | A4 |
| Citation | OSCOLA (Oxford Standard for Citation of Legal Authorities) |
| Date | DD Month YYYY (e.g., 10 March 2026) |
| Spelling | UK English throughout |
| Numbering | Clause 1, 1.1, 1.1.1 or (a), (b), (c) |
| Margins | 2.54cm (1″) all sides |
| Font | Times New Roman 12pt body text |

## Register and Voice
- "Must" is preferred over "shall" unless reproducing legacy wording or prescribed text
- Use direct verbs: `provides`, `states`, `requires`, `permits`, `prohibits`
- Prefer active voice where the actor matters; use passive voice only when the actor is unknown, obvious, or legally irrelevant
- Avoid colloquialisms, conversational hedges, and US-specific idiom
- Avoid archaic legalese such as `hereinafter`, `aforementioned`, `whereas` in advisory prose unless reproducing a formal board-resolution style

## Numbering System

### Contracts, policies, and structured corporate documents
```
Clause 1
1.1
1.1.1
(a)
(i)
```

### Advice memoranda and submissions
```
1.
1.1
1.2
2.
```

### Numbering rules
- Do not mix `Section` and `Clause` in the same UK document
- Nested numbering must remain decimal unless the governing form requires letters or Roman numerals
- Use `(a), (b), (c)` for subparagraphs and `(i), (ii), (iii)` below that
- If a level has only one child item, consider whether the sublevel is necessary at all

## Defined Terms
- Define terms sparingly and only where repeated use improves clarity
- Capitalise defined terms consistently after definition
- Prefer formulae such as `ABC Ltd (the Company)` or `the Financial Conduct Authority (the FCA)`
- Do not create multiple defined terms for the same concept
- If the document is advice rather than transactional drafting, prefer natural repetition over over-defined prose

## Document Structure Guidance

### Memorandum / advice note
1. Heading block
2. Question presented or instructions received
3. Short answer / executive summary
4. Facts or assumptions
5. Applicable law
6. Analysis
7. Conclusion
8. Limitations / qualifications where needed

### Court submission / skeleton argument
1. Introduction
2. Issues
3. Facts or procedural background
4. Legal framework
5. Submissions
6. Relief sought / conclusion

### Corporate document
1. Title
2. Recitals or background if needed
3. Operative clauses / resolutions
4. Signature or certification block

## OSCOLA Citation

### Cases
Format:
```text
Party v Party [Year] Court Reference
```

Examples:
- `R v Smith [2023] UKSC 45`
- `Donoghue v Stevenson [1932] AC 562`

Rules:
- Use `v`, not `v.`
- Case names are normally italicised in final formatted output
- Use neutral citations where available
- Pinpoints should follow the citation in brackets where needed, following OSCOLA style

### Statutes
Format:
```text
Act Name Year, s Section
```

Examples:
- `Companies Act 2006, s 171`
- `Data Protection Act 2018, s 170`

Rules:
- Use `s` / `ss` for sections and `sch` / `para` where applicable
- Do not add US-style reporter references or title numbers

### Secondary Sources
- OSCOLA is primarily footnote-based
- Avoid Bluebook-style inline citation strings
- For books, articles, and reports, use standard OSCOLA footnote patterns
- In prose, a short descriptive reference is acceptable if full citation appears in a footnote

## Dates, Names, and Institutional References
- Use `10 March 2026`, not `March 10, 2026`
- Use full institutional name on first reference, then a defined short form if needed
- For courts and regulators, preserve official UK names and capitalisation

## Page Setup Summary
- Use A4 paper
- Use the parameters in `../../output-formatter/references/page-setup-en-uk.md`
- Default body font is Times New Roman 12pt
- Use justified alignment unless a court rule or house style requires left alignment

## Common Drafting Corrections
| Avoid | Prefer |
|---|---|
| `section` (in UK operative drafting) | `clause` |
| `v.` | `v` |
| `shall` | `must` |
| `Exhibit` | `Schedule` or `Appendix` |
| `attorney` | `solicitor`, `counsel`, or role-specific term |
| `pursuant to` | `under` or `in accordance with` |

## Quality Checklist
- UK spelling is consistent throughout
- Numbering uses UK conventions consistently
- OSCOLA citation is used rather than Bluebook
- `v` is used without a period
- Dates are in `DD Month YYYY` format
- Defined terms are used consistently and sparingly
- The document sounds formal, clear, and modern rather than archaic
